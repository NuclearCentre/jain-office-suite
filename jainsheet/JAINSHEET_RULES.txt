BAT FILE TECHNICAL RULES (CRITICAL -- DO NOT CHANGE)
------------------------------------------------------
These 4 patterns are CONFIRMED WORKING and must exist in every version
of jainsheet_deploy.bat. Never remove or alter them.

1. SELF-RELAUNCH at top of file (keeps window open -- NEVER remove):
     if "%1"=="RELAUNCHED" goto :MAIN
     start "JainSheet Deploy" /WAIT cmd /k "%~f0" RELAUNCHED
     exit
   Why: shortcuts launch cmd which closes instantly on any error.
   The /k flag keeps the window open. Without this 3-line block the
   window disappears before you can read any error message.

2. SAFE DIRECTORY WILDCARD (fixes "dubious ownership" Git error):
     git config --global --add safe.directory "*"
   Why: Git refuses to run in folders owned by a different Windows user
   (TrustedInstaller). The wildcard permanently fixes this for all
   folders. Never replace with a specific path.

3. GIT IDENTITY (fixes "commit failed -- check user.name" error):
     git config --global user.name "NuclearCentre"
     git config --global user.email "admin@jainsheet.com"
   Why: Git requires name+email for every commit. Not set by default
   on Windows. Setting inside the .bat ensures they are always present.

4. FORCE WORKING DIRECTORY before every git command:
     cd /d D:\JainSheet
     git -C "D:\JainSheet" <command>
   Why: shortcuts start from C:\Windows\System32. Even after cd, some
   git calls inherit the wrong directory. Using BOTH cd AND git -C
   together guarantees the correct folder every single time.

ARCHITECTURE RULES
------------------
- Save system: renderer.js sends write-file IPC -> main.js writes to disk
  -> replies write-file-done
- XLSX export uses SheetJS (xlsx npm package) with cellStyles:true --
  exported as base64 buffer, written by main.js using isBuffer:true flag
- trigger-saveas: renderer sends this when Ctrl+S pressed with no file
  open -- main.js MUST have ipcMain.on('trigger-saveas') that opens
  the Save As dialog and sends menu-saveas back to renderer
- Zoom uses CSS transform scale() on #grid-wrap div inside
  #grid-container -- never scale the container itself
- Arrow key navigation uses a hidden #_focusTrap div (tabIndex=0) --
  NEVER attach arrow key navigation to #grid-container (scrollable div
  steals arrow keys for browser scroll)
- After every selectCell() call, grabFocus() must be called to keep
  keyboard focus on _focusTrap
- Formula mode cell click: uses .formula-sel class (dashed blue) only --
  never .sel or .inr -- extSel() must be BLOCKED during formula mode
- Settings (page size, orientation, zoom) are stored in JS variables --
  NOT localStorage
- Multi-sheet state stored in sheets[] array with activeSheet index
- renderer.js exposes ALL public functions via window.X = X at the
  bottom of the file -- if you add a new function that index.html
  buttons call, you MUST add it to that window exports block or the
  button will silently do nothing

- colName(c): single letter for c<26, two letters (AA-AZ) for c>=26
- parseRef() strips ALL $ before parsing -- absolute refs $A$1/$A1/A$1 work
- hiddenRows / hiddenCols: Sets stored per-sheet and saved/restored in JSON
- tabColor: stored per-sheet, rendered as colored dot on tab
- Distinct error values: #DIV/0! #VALUE! #REF! #NAME? (not all #ERR)

- renderCell() now also handles mergedCells -- checks isMergeHidden first

- XLSX open: reads cell.f (formula), cell.w (formatted value), all sheets
- XLSX export: writes formatting + formulas + merges via SheetJS cellStyles:true
- loadState() restores mergedCells per sheet from v2 JSON
- getState() saves mergedCells in v2 JSON

MERGE & CENTER -- IMPLEMENTATION DETAILS
-----------------------------------------
- mergedCells: object per sheet, key = master cell ID e.g. "A1"
  value = {r1, c1, r2, c2} defining the merged rectangle
- isMergeHidden(r,c): returns true if cell is a slave inside a merge
- getMasterCell(r,c): returns {r,c} of master if cell is inside any merge
- mergeToggle(): if master already exists -> unmerge; else -> merge selection
- unmergeRange(masterId): deletes merge, restores individual cells
- renderCell: hides slave cells (display:none), applies colspan/rowspan on master
- Clicking a slave cell redirects to master via getMasterCell() in startSel
- Merges saved in getState() and restored in loadState()
- Merges exported as ws['!merges'] in exportXlsx
- Merges imported from ws['!merges'] in importXlsx

COLOR PICKER -- IMPLEMENTATION DETAILS
---------------------------------------
- openFontColor(btnEl) / openFillColor(btnEl) -- buttons pass `this`
- showColorPicker(anchorEl, target): creates popup div anchored below button
- COLOR_PALETTE: array of 50 hex colors (5 rows x 10 cols)
- Swatches scale up on hover; click applies color instantly
- Native <input type="color"> for unlimited custom colors
- None button clears the color
- _colorPickerTarget: 'font' or 'fill'
- applyColorPickerValue(hex): calls applyStyleProp('color') or applyStyleProp('bg')
- closeColorPicker(): removes popup, removes outside-click listener
- Picker positions below button, flips upward if near bottom of screen

XLSX ROUND-TRIP -- IMPLEMENTATION DETAILS
------------------------------------------
- main.js reads .xlsx/.xls as base64 (fs.readFileSync().toString('base64'))
  NOT utf8 -- binary files corrupted if read as utf8
- importXlsx(base64Content): SheetJS XLSX.read() with cellStyles+cellFormula+cellDates
  Reads: formulas (cell.f), values, number formats, bold/italic/color/fill,
  alignment, borders (all 4 sides with style+color), column widths, merges
- exportXlsx(filePath): builds ws cell-by-cell (not aoa_to_sheet)
  Writes: cell.f for formulas, cell.v for values, cell.s for full style object,
  ws['!merges'] for merged regions, ws['!cols'] for column widths
- XLSX.write() called with {cellStyles: true}
